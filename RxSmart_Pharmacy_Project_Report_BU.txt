Abstract
Modern pharmaceutical distribution points and retail medicine portals manage heavy transactional counts, batch-specific drug expirations, flexible sales tax computations, and regulatory client notifications. Traditional offline desktop databases introduce critical operational risks, including localized file corruption, hardware rigidity, complex backup protocols, and complete data loss on host machine failures. To overcome these core limitations, this project introduces "RxSmart Pharmacy Management System", a responsive, resilient, and highly secure web-based POS platform designed for retail pharmacy counters.

Engineered with a React and Vite frontend framework and a Node.js/Express backend API gateway, RxSmart runs a lightweight JSON database layer implemented with safe, concurrent write controls on Windows servers. Key technical innovations include a Client-Side local storage sync engine designed to automatically back up critical states (medicines list, transaction logs, notifications) directly inside the browser's persistent cache, shielding pharmacy databases against cloud resets or physical server wipes. The system integrates a secure SMTP invoice dispatch tunnel via Nodemailer, enabling cashiers to manually or automatically email HTML receipts to clients. It also features background inventory scanners running on set schedules to detect upcoming drug batch expiries and low stock levels. Wrapped in a stunning, modern dark/light design system powered by Vanilla HSL CSS and responsive glassmorphic cards, RxSmart delivers a reliable, high-performance checkout experience that ensures patient trust and business continuity.

1. Introduction
1.1 Overview of Pharmacy Management Industry
The pharmaceutical retail and distribution sector is a highly regulated, high-throughput component of global healthcare logistics. Unlike standard retail operations, pharmacies must manage unique constraints, including the storage and tracking of specific chemical formulations, batch-specific drug expirations, precise sales tax (GSTIN) logs, and strict inventory level audits to prevent critical medication stockouts. Operationally, cashiers are required to process billing rapidly at front counters, while managers must continuously cross-reference batch lists with shelf stock to identify and purge expired medicines before they are sold to patients.
Historically, pharmacy systems relied on physical ledger books or offline, desktop-bound point-of-sale (POS) systems. These offline setups suffer from severe architectural limitations: they lock the store's data onto a single physical hard drive, lack mobile accessibility for managers working off-site, and require manual, complex database backup procedures. A minor operating system failure or physical disk corruption can wipe out years of business logs. With modern web APIs and cloud hosting, transitioning to thin-client web applications represents a major operational improvement, offering cross-device access, automated warning systems, and centralized logs with minimal local overhead.
1.2 Problem Statement
Despite general POS platforms becoming widely available on the web, small-to-medium retail pharmacies remain underserved by tools that adequately safeguard their operational logs against sudden data loss. Web-based SaaS platforms depend heavily on constant network availability and remote server database stability; a server crash or database wipe can instantly paralyze a busy sales counter. Additionally, manual invoice receipt archiving results in lost paper trails and delayed record updates.
( A Store Owner Worried About Drug Expiries & Lost Receipts )
The core challenges identified in the existing pharmacy management systems include:
Lack of a Unified POS Flow: A fractured interface requiring separate applications for inventory entry, customer contact lookup, and sales receipt mailing.
Local Database Vulnerability: Cloud server restarts or host database resets can clear active tables. If a server resets in a pure web-app environment, cashiers lose access to pricing catalogs and history.
Regulatory Expiry Blindspots: Manual check of batch sheets is highly error-prone, creating legal and health risks if expired medicines are sold.
Lack of Digital Delivery Tunnels: Offline registers fail to provide digital records, leading to patient disputes and lost invoice receipts.
1.3 Proposed Solution: RxSmart
RxSmart was engineered as a highly resilient, responsive full-stack pharmacy ecosystem that addresses these issues. Operating as a web application with a React client and Node.js/Express API server, RxSmart simplifies inventory tracking and front-counter billing.
To address server reliability concerns, RxSmart integrates a Client-Side local storage sync engine (Resilience Engine). This engine automatically maintains duplicate records of current inventories, transactions, and notification boards directly inside the browser's persistent cache. If a backend database reset occurs, the pharmacy manager can immediately restore the backend files using a single click from the settings dashboard, preventing sales interruptions. The checkout module validates stock levels in real-time, generates dynamic receipts branded with the shop's customized parameters, and triggers automated invoice dispatches directly to the client's email via Nodemailer.
(RxSmart An Ultimate Solution)
1.4 Objectives of the Project
The primary technical and operational objectives of the RxSmart Pharmacy Management System are:
To develop a responsive web application utilizing a decoupled client-server architecture (React/Vite & Node.js/Express).
To implement a local storage sync engine that caches data in browser memory to secure logs against server resets.
To construct a secure Nodemailer SMTP tunnel for delivering HTML invoice receipts to patient inboxes at checkout.
To design a customizable, double-themed (dark/light) user interface utilizing Vanilla HSL CSS and frosted glass panel blurs.
To establish an automated background scanning service that continuously monitors batch expiration logs and low-stock levels.
To support custom store registration parameters (GSTIN, name, address, contact) that dynamically bind to cashier invoice headers.
1.5 Scope of the Project
The project scope encompasses all critical daily processes within a retail or wholesale pharmacy shop. It provides a secure signup/login portal protected by JWT and email-delivered OTP verification. The application includes a dashboard with real-time sales metrics and expiry logs, an inventory database for managing medicine batch entries, prices, and quantities, a POS checkout module, and an invoice archive with resend capabilities.

2. System Architecture & Tech Stack
2.1 High-Level Architecture Framework
RxSmart is structured on a modern, decoupled client-server framework. The frontend presentation layer handles the user experience, navigation routes, and local backups, communicating with the Node.js API server via asynchronous HTTP requests. The backend server manages authentication middleware, inventory updates, file writes, and the Nodemailer SMTP dispatch system. A lightweight JSON database handles persistent storage.
2.2 Frontend Technologies
The user portal is built on React.js and Vite to deliver high performance, responsive pages, and rapid build speeds:
React.js (Single Page Application): Used for structured layouts, state hooks, and component encapsulation.
Vite bundler: Ensures immediate page rendering and instant hot module replacement during development.
Vanilla HSL CSS: Configured with custom HSL color properties, glassmorphic variables, and media query grids for responsive UI styling.
Browser Storage APIs: Utilized to maintain backup sync stores for off-network or database recovery tasks.

const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:5000/api';

function getHeaders() {
  const token = localStorage.getItem('pharmacy_token');
  const headers = { 'Content-Type': 'application/json' };
  if (token) {
    headers['Authorization'] = `Bearer ${token}`;
  }
  return headers;
}

async function request(endpoint, options = {}) {
  const url = `${API_BASE_URL}${endpoint}`;
  const config = { ...options, headers: { ...getHeaders(), ...options.headers } };
  try {
    const response = await fetch(url, config);
    const data = await response.json();
    if (!response.ok) throw new Error(data.message || 'Request failed');
    return data;
  } catch (error) {
    console.error(`[API Error] ${endpoint}:`, error);
    throw error;
  }
}
2.3 Backend Technologies
The server application handles request validation, authentication checks, and email delivery:
Node.js & Express.js: The core runtime environment and router, executing routing functions.
BcryptJS & JSON Web Tokens (JWT): Handles password encryption, validating credentials against hashed strings.
Nodemailer: Implements SMTP connections to secure mail servers, building and transmitting HTML invoice templates.
2.4 Database Technologies
To enable simple backups and deployment, RxSmart uses a file-based JSON database structure. The database collections reside in `backend/data/` as structured JSON arrays: `users.json`, `medicines.json`, `bills.json`, and `notifications.json`. To prevent file write corruptions on concurrent Windows processes, the database driver uses synchronized file locks during operations.

4. Advanced Feature Implementation
4.1 Browser Local Storage Sync Engine (Resilience Engine)
To protect pharmacy data from cloud server failures or database resets, RxSmart implements a client-side database backup engine. During active usage, the React frontend automatically stores copies of local inventories, bills, and alerts within the browser's persistent `localStorage` and `sessionStorage`. If the server database is reset, the manager can restore all tables using the setting controls, syncing local data back to the server.
4.2 Nodemailer SMTP Invoice Dispatch Tunnel
RxSmart replaces paper receipts with an email delivery system powered by Nodemailer and SMTP. When a cashier completes a sale, the backend generates a styled HTML invoice containing the transaction details (drugs, batch IDs, prices, discounts, GST, total) and shop parameters (name, contact, address). It then emails the invoice directly to the customer.
Additionally, cashiers can edit or update the customer's email address on-the-fly from the receipt preview modal, which updates the address in the database and delivers the email.
4.3 Automated Low-Stock & Expiry Notification Scanners
To maintain inventory control, RxSmart uses background scanners to track stock health. The backend runs checking routines at startup, on manual request, and at six-hour intervals. It flags medicine batches with stock counts below 20 units or expiration dates within 30 days, logging notifications to alert the manager.

6.2 REST API Endpoint Specifications
The backend API exposes the following endpoints. All protected routes require a JWT authorization token.
Authentication Endpoints
POST /api/auth/signup - Creates a new user, hashes password, assigns shop parameters.
POST /api/auth/login - Verifies credentials, generates JWT access tokens.
POST /api/auth/send-otp - Sends verification OTP to the user's email.
POST /api/auth/restore - Rebuilds database states from browser backup payloads.
Medicine Inventory Endpoints
GET /api/medicines - Retrieves the list of cataloged medicines (Protected).
POST /api/medicines - Creates a new medicine batch entry (Protected).
PUT /api/medicines/:id - Updates details of a medicine batch (Protected).
DELETE /api/medicines/:id - Removes a medicine batch from the database (Protected).
POST /api/medicines/scan - Triggers an on-demand stock and expiry check (Protected).
Billing & Invoice Endpoints
POST /api/billing - Creates a billing record and decrements stock levels (Protected).
GET /api/billing - Retrieves the invoice transaction archive (Protected).
POST /api/billing/:id/send-email - Emails a transaction receipt to the customer (Protected).
